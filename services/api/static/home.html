About the Project

SecPurityAI is an open-source intrusion detection and cybersecurity research platform developed by Praharsh Pulla. It combines machine learning, deep learning, and reasoning models with explainability, governance, and privacy-by-design principles.

The project’s purpose is to deliver a comprehensive tool that serves a dual focus: operational security teams (SOCs) requiring real-time, actionable alerts, and researchers exploring advanced AI techniques for threat detection. Its unique differentiators include integration of multi-modal AI models, federated learning for privacy preservation, and a strong emphasis on transparency and compliance with emerging AI governance frameworks.

Core Features

🧠 Hybrid AI Detection

Integrates rule-based systems, machine learning (ML) and deep neural networks (DNN), large language model (LLM) reasoning, and graph anomaly detection. This multi-layered approach enables comprehensive threat identification by combining signature-based, behavioral, contextual, and relational data analyses for robust and adaptive intrusion detection.

For example, it can detect both known malware signatures and novel attack patterns by correlating network flows with user behavior, significantly reducing false positives.

⚔️ Red vs Blue Simulation

Emulates adversary tactics, techniques, and procedures (TTPs) within a safe cyber battle arena, allowing defenders to test and improve their strategies in realistic attack scenarios. This simulation fosters continuous defense enhancement by modeling attacker behaviors and enabling proactive security posture adjustments.

This enables SOC teams to practice incident response and validate detection rules against simulated advanced persistent threats (APTs) before real-world deployment.

📜 Explainability & Governance

Provides detailed decision logs, model transparency through model cards, and alignment with compliance frameworks such as NIST AI Risk Management Framework. This ensures auditability, accountability, and trustworthiness in AI-driven cybersecurity operations.

Users can trace why an alert was triggered, facilitating regulatory audits and building confidence in automated systems.

🔒 Privacy by Design

Implements federated learning techniques to train models across decentralized data sources without exposing raw data, combined with differential privacy mechanisms to protect sensitive information. This design ensures compliance with privacy regulations while enabling collaborative threat intelligence.

This allows organizations to collaboratively improve detection models without sharing proprietary or sensitive data, maintaining compliance with GDPR and other frameworks.

📈 Continuous Learning

Automatically adapts to emerging threats, including zero-day vulnerabilities, by incorporating new data into ongoing model retraining processes. This continuous learning pipeline maintains detection efficacy and reduces manual intervention, enabling timely responses to evolving attack landscapes.

For instance, the system can quickly learn from new phishing campaigns or malware variants, minimizing the window of exposure.

Architecture Snapshot

Data → AI Pipelines (ML/DL/LLM) → Policy Engine → Blue/Red Agents → Dashboard & Explainability Layer.

The Data component collects and preprocesses logs, network traffic, and endpoint telemetry. The AI Pipelines apply machine learning, deep learning, and large language models to analyze and detect anomalies or threats. The Policy Engine enforces security policies and compliance rules, guiding automated responses. Blue/Red Agents simulate defensive and offensive operations for testing and training. Finally, the Dashboard & Explainability Layer provides visualization, alerts, and transparent explanations for all detection and decision-making processes.

Planned Roadmap

  • Phase 0: Foundations – Establish core data ingestion, baseline detection models, and initial infrastructure setup.
  • Phase 1: RAG + Policy Enforcement – Integrate Retrieval-Augmented Generation for context-aware reasoning and implement policy enforcement mechanisms.
  • Phase 2: Red vs Blue MVP – Develop minimum viable product for adversarial simulation with basic attacker and defender agents.
  • Phase 3: Graph Intelligence – Incorporate graph-based analytics to detect relational anomalies and lateral movement within networks.
  • Phase 4: Privacy & Federation – Implement federated learning and privacy-preserving technologies to enable collaborative model training.
  • Phase 5: Deception Systems – Deploy honeypots and decoys integrated with AI to mislead and analyze attackers.
  • Phase 6: Hardening & Publication – Finalize security hardening, documentation, and release for public and enterprise adoption.

Future Directions

🤖 LLM Reasoning

Integrating large language models for threat analysis and decision support. This will enable natural language understanding of alerts and automated incident summarization, improving analyst productivity and situational awareness.

🧩 ML & Deep Learning

Pipeline improvements for classification, anomaly detection, and adaptive learning stages. Focused on enhancing accuracy, reducing false positives, and enabling real-time model updates to keep pace with evolving threat landscapes.

🛠️ Playground

Interactive sandbox for testing and experimenting with models and simulations. This environment will allow researchers and practitioners to prototype new detection algorithms and simulate attack-defense scenarios safely.

🌐 Multi-Modal Detection

Extending to logs, images, and deepfake analysis. This expansion aims to detect sophisticated attacks that use multimedia channels, including social engineering via manipulated media.

🔮 Quantum-Safe Security

Exploring future cryptography and resilience mechanisms. Preparing for the post-quantum era by researching encryption algorithms and defensive techniques resistant to quantum computing threats.
